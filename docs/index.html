Step-by-step walkthrough of the CPU-only, open-source sequencing pipeline in this repository. Adjust the settings panel below and every command block updates live — copy and paste into your terminal for real runs or re-runs with different paths, panels, and samples.

Overview — data types and workflows

Genomic analysis starts from one of two measurement types. Both converge on VCF for downstream annotation and reporting.

Quickstart — synthetic demo

The fastest way to see the pipeline end-to-end: a 2-sample synthetic demo (no real patient data). These commands are fixed — not tied to the settings panel. 

conda env create -f envs/environment.yml
conda activate variant-pipeline

cd test_case
./run_demo.sh
python check_demo.py results

Expected: case1.report.html shows one Pathogenic MYBPC3 hit; control1 does not. See test_case/README.md on GitHub.

Walkthrough — stage by stage

Run stages in order for a single sample, or skip to full cohort run. Each block includes the shared export lines your scripts read from pipeline/00_config.sh.

Setup

Environment

Create the conda environment once. Includes bwa, samtools, bcftools, GATK4, snpEff, fastp.

01

Prepare reference

Build BWA index, samtools .fai, GATK sequence dictionary, and tabix-index the ClinVar VCF. Run once per reference FASTA — skipped on subsequent runs if indexes exist.

02

Align reads

fastp QC/trim → bwa mem alignment → samtools sort → samtools markdup → index. Output: {sample}.markdup.bam.

03

Call variants

Two independent callers: bcftools (primary, pileup-based) and GATK4 HaplotypeCaller (secondary, local re-assembly). Both produce hard-filtered PASS-only VCFs.

04

Annotate variants

snpEff gene/consequence (if database installed) + ClinVar clinical significance via bcftools annotate. Optional Ensembl VEP cross-check if vep is on PATH.

05

Filter pathogenic calls

Extract ClinVar Pathogenic / Likely pathogenic variants into per-sample JSONL, tagged with case or control type for cohort aggregation.

07

Generate HTML report

Self-contained interactive HTML report: pathogenic/VUS calls, panel-gene highlights, sortable/filterable table. Panel name appears in the report title.

06

Aggregate cohort (after all samples)

Group pathogenic calls across all samples: count how many case vs control samples carry each variant. Run once after every sample has completed stage 05.

Full cohort run

run_pipeline.sh runs stages 01–07 for every row in a tab-separated manifest, then aggregates pathogenic calls. Equivalent to running the walkthrough stages in a loop.

Manifest columns: sample_id   case|control   R1.fastq.gz   R2.fastq.gz

Roadmap and limitations

  • Roadmap: Array/genotyping path (IDAT/CEL → PLINK → imputation → VCF)
  • Roadmap: Pluggable alternate callers (e.g. DeepVariant) via config
  • Roadmap: Nextflow/Snakemake orchestration with structured run provenance
  • Linear GRCh38 only — no DRAGEN pangenome graph mode equivalent
  • Annotation via public ClinVar + optional snpEff/VEP — not Illumina Nirvana
  • Provided for technical/educational use — not validated for clinical diagnostics

Swap panels by changing PANEL_GENES, PANEL_BED, and PANEL_NAME — see panels/ on GitHub.
